# Xepha Vision

Xepha is a local-first intelligence layer for software projects.

Its job is to preserve project continuity outside the chat window by modeling
decisions, events, commits, issues, pull requests, deployments, tests,
architecture choices, and their relationships.

## Product Thesis

Current AI coding tools are powerful, but large projects still lose continuity.
Agents rebuild context repeatedly, compress history into weak summaries, and
forget why decisions were made.

Xepha treats project knowledge as structured, queryable intelligence instead of
linear text.

## MVP Claim

The first MVP should prove that project knowledge can be structured outside the
chat and reused by agents with lower token cost and better continuity.

## Non-Goals

- Xepha is not another LLM.
- Xepha is not another coding assistant.
- Xepha is not a markdown documentation format.
- Xepha is not a GraphRAG wrapper.
- Xepha is not an MCP replacement.

## Direction

- Local first
- Open source
- Modular
- Developer first
- Agent friendly
- Observable
- Composable
